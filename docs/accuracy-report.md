## 2. 건기식 성분 해석율 (Resolve Rate)

### 측정 방법

- 실제 앱에서 들어올 법한 성분명 표현 **74개** 수동 구성
  - 정답 있는 케이스 (매핑 가능해야 하는 것): 62개
  - 정답 없는 케이스 (매핑 되면 안 되는 것): 12개
- 스크립트: `supplement_recognition/scripts/evaluate_resolve_rate.py`
- 대상 DB: `click_backend_db.supplement_entities` (33종)

### 결과

| 지표 | 수치 |
|---|---|
| TP — 정답 있고 올바르게 매핑 | 41/62 = **66.1%** |
| FN — 정답 있는데 못 찾음 | 21/62 = **33.9%** |
| TN — 비매핑 케이스 정상 처리 | 12/12 = **100%** |
| FP — 없어야 하는데 오매핑 | 0/12 = **0%** |
| **Precision** | **100.0%** |
| **Recall** | **66.1%** |
| **F1 Score** | **79.6%** |

> **Precision 100%** = 매핑됐을 때 틀린 적 없음 — 오탐(FP) 0건  
> **Recall 66%** = 표현 방식이 다르면 (별칭, 속칭, 약어) 못 찾음

### FN 21건 분류

| 유형 | 예시 | 건수 |
|---|---|---|
| 별칭·속칭 미등록 | 오메가3, 알티지 오메가3, 유산균, 홍삼, 코큐텐 | 10건 |
| 영문 약어 | CoQ10, EGCG, GLA, CLA | 4건 |
| 활성 성분명 (상위 개념 필요) | 실리마린→밀크씨슬, 알리신→마늘, 이소플라본→대두, 강황→울금 | 4건 |
| 기타 표기 차이 | 카르두스 마리아누스, 엘카르니틴 | 3건 |

**개선 방향:** `supplement_entities`에 alias 테이블 추가 또는 `supplement_name_en` 컬럼에 약어/별칭 등록

## 5. False Negative 발생 경로

안전 관련 서비스에서 가장 치명적인 지표입니다.

| FN 발생 경로 | 현재 상태 | 규모 |
|---|---|---|
| 상호작용 DB 미등재 약물과의 조합 | 발생 가능 | 41종 약물 관련 조합 전체 |
| 이미지 인식 실패 → 잘못된 성분 조회 | 발생 가능 | 전체의 25.9% (7/27) |
| 성분 해석 실패 → 조회 자체 누락 | 발생 가능 | 성분 표현의 33.9% |
| 위험 키워드 없는 문장 → safe로 fallback | 발생 가능 | 별도 측정 필요 |

**현재 FN 정량화 상태:** 전문가 기반 정답셋(ground truth) 미구축으로 민감도/특이도 직접 계산 불가.  
DB 미등재 약물(41종) 조합은 시스템이 "분석 불가"로 명시적 경고를 표시하도록 처리되어 있어, 묵시적 safe 반환은 발생하지 않음.

## 6. 전체 파이프라인 정확도 요약

```
[이미지] ─→ [Gemini 추출] ─→ [DB 매칭] ─→ [성분 해석] ─→ [상호작용 조회]
             100% 성공         74.1% 정확     66.1% Recall    36.9% 제품 커버
```

파이프라인 전 구간이 정상 동작할 때의 End-to-End 기대 성공률:

```
0.741 (이미지 인식) × 0.661 (성분 해석) × 0.369 (DB 커버) ≈ 18%
```

> 단, 이미지 인식과 성분 해석은 독립 단계이며, DB 커버리지는 제품 종류에 따라 달라짐.  
> 프로바이오틱스·인삼·오메가-3 등 시중 유통 비중이 높은 성분은 실질 커버리지가 36.9%보다 높음.
