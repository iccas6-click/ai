## 5. False Negative 발생 경로

안전 관련 서비스에서 가장 치명적인 지표입니다.

| FN 발생 경로 | 현재 상태 | 규모 |
|---|---|---|
| 상호작용 DB 미등재 약물과의 조합 | 발생 가능 | 41종 약물 관련 조합 전체 |
| 이미지 인식 실패 → 잘못된 성분 조회 | 발생 가능 | 전체의 25.9% (7/27) |
| 성분 해석 실패 → 조회 자체 누락 | 발생 가능 | 성분 표현의 33.9% |
| 위험 키워드 없는 문장 → safe로 fallback | 발생 가능 | 별도 측정 필요 |

**현재 FN 정량화 상태:** 전문가 기반 정답셋(ground truth) 미구축으로 민감도/특이도 직접 계산 불가.  
DB 미등재 약물(41종) 조합은 시스템이 "분석 불가"로 명시적 경고를 표시하도록 처리되어 있어, 묵시적 safe 반환은 발생하지 않음.

## 6. 전체 파이프라인 정확도 요약

```
[이미지] ─→ [Gemini 추출] ─→ [DB 매칭] ─→ [성분 해석] ─→ [상호작용 조회]
             100% 성공         80.0% 정확     66.1% Recall    36.9% 제품 커버
```

파이프라인 전 구간이 정상 동작할 때의 End-to-End 기대 성공률:

```
0.800 (이미지 인식) × 0.661 (성분 해석) × 0.369 (DB 커버) ≈ 19%
```

> 단, 이미지 인식과 성분 해석은 독립 단계이며, DB 커버리지는 제품 종류에 따라 달라짐.  
> 프로바이오틱스·인삼·오메가-3 등 시중 유통 비중이 높은 성분은 실질 커버리지가 36.9%보다 높음.

## 7. 처방전/약봉투 인식 정확도

### 측정 방법

- 테스트 이미지: `pill_recognition/datasets/evaluation/real-smartphone/images/` 내 20장
- 처방전, 복약안내문, 약봉투 혼합
- 스크립트: `scripts/evaluate_prescription_accuracy.py`
- 판정 기준: `match_type`이 `not_found` / `llm_only`가 아닌 경우 DB 매칭 성공으로 판정

### 파이프라인

```
이미지
  → [1] Gemini Vision — 약품명 + 성분명 후보 구조화 추출
  → [2] 백엔드 DB 매칭 (우선순위)
       ① official_drug_products (상세 약품 정보 포함) — 현재 미적재
       ② pill_products + pill_product_ingredients + canonical_drug_entities
       ③ DB 미등재 → Gemini 성분명 2차 추출 → canonical_drug_entities 매핑
  → [3] 결과 반환 (제품명, 성분, 복용법, needs_confirmation)
```

### 결과

| 지표 | 수치 | 비고 |
|---|---|---|
| 테스트 이미지 수 | 20장 | 경구 의약품 19장 + 안약 1장 |
| 약품명 추출 성공 (경구) | 19/19 = **100%** | 안약 이미지 제외 |
| 추출된 경구 약품명 수 | **95건** | 안약 3건 DB 매칭 제외 |
| 제품명 DB 직접 매칭 | 5/95 = **5.3%** | `pill_products`에 실제 등재된 경우 |
| 성분명 canonical 매칭 | 90/95 = **94.7%** | Gemini 성분 추출 → `canonical_drug_entities` 경유 |
| 완전 미매칭 | 0/95 = **0%** | |
| 안약 이미지 처리 | 1장 / 3건 | `eye_drop` 타입으로 분류 → DB 조회 스킵 |

> **해석:** 처방전 약품의 94.7%는 `pill_products`에 제품명이 없어 Gemini가 성분명을 직접 추출한 뒤 `canonical_drug_entities`에서 매핑됩니다. `pill_products` DB가 처방약을 충분히 커버하지 못하고 있으며, 상호작용 분석은 Gemini의 성분 추출 정확도에 의존합니다.

### 문서 유형 분포

| 유형 | 건수 |
|---|---|
| 복약안내문 (medication_guide) | 11건 |
| 처방전 (prescription) | 4건 |
| 약봉투 (medicine_bag) | 4건 |
| 안약 (eye_drop) | 1건 |

### 매칭 방식 분포

| match_type | 설명 |
|---|---|
| `ingredient_text` | Gemini가 추출한 성분명 → `canonical_drug_entities` 직접 매핑 (다수) |
| `legacy_product_table` | `pill_products` 제품명 직접 매칭 (일부) |
| `llm_product_ingredient_candidate` | DB 미등재 → Gemini 2차 성분 추출 후 매핑 (1건: 비알코시럽) |

> `official_drug_products` 테이블이 현재 미적재 상태로 ①단계는 동작하지 않음.  
> 적재 시 제품 이미지, 효능, 부작용, 상호작용 원문 등 추가 정보 제공 가능.

  2. **처방전 약물 커버리지** — 처방전 20장에서 추출한 78종 성분이 `canonical_drug_entities`에 매핑되는 비율
  3. **상호작용 히트율** — 매핑된 처방전 약물 × 33종 건기식 조합 중 실제 상호작용 존재 비율

### 처방전 약물 매핑률 14.1%에 대한 해석

`canonical_drug_entities`(178종)는 건기식과의 상호작용이 문헌에 보고된 약물만 포함합니다.  
처방전에서 흔히 나타나는 **항생제, 항히스타민제, 소화기약, 안과약** 등은 건기식 상호작용 데이터가 없어 의도적으로 미등재 상태입니다.  
**이는 시스템 버그가 아닌 DB 설계 범위**이며, 조회 자체는 "분석 불가"로 명시적 응답을 반환합니다.

매핑된 11종은 모두 **심혈관계, 정신과, 당뇨병, 소화기** 약물로 건기식 상호작용 위험이 실제로 알려진 클래스입니다.
